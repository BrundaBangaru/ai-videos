Deploy Free
// host this platform for free · step-by-step guide
✅ This entire platform is a single HTML file. No server needed. No backend. No database. Host it anywhere static files are supported — all 100% free.
1
Get the File
Download index.html from this page. This is your entire app — one file, no dependencies, no build step needed.

Alternatively, create a new file locally and paste all the code into it.
2
Create GitHub Repo
1. Go to github.com → Sign up free
2. Click "New repository" → name it astra-studio
3. Set to Public
4. Upload your index.html file
5. Commit changes
3A
Deploy via GitHub Pages
In your repo: Settings → Pages
Source: Deploy from branch
Branch: main → / (root)
Click Save.

URL: username.github.io/astra-studio
✅ Free forever. Custom domain supported.
3B
Deploy via Vercel
1. Go to vercel.com → Sign up with GitHub
2. "Import Project" → select your repo
3. Click Deploy (no config needed)

URL: astra-studio.vercel.app
✅ Free tier: 100GB bandwidth/month. SSL included. Auto-deploy on git push.
3C
Deploy via Netlify
1. Go to netlify.com → Sign up free
2. Drag & drop your index.html into the deploy area
3. Done in 10 seconds.

URL: random-name.netlify.app
✅ Free tier: 100GB/month. Instant deploy. No git needed.
4
Get HF Token (Optional)
Free HF tokens give higher rate limits:
1. Go to huggingface.co → Create account
2. Settings → Access Tokens → New token
3. Role: Read is enough
4. Paste into the token bar at the top

⚠ Never commit your token to GitHub.
5
Custom Domain
Want yourdomain.com?

Free options:
• Freenom — free .tk/.ml domains
• js.org — free subdomain for devs
• Cloudflare Pages — free hosting + free SSL

Connect your domain to Vercel/Netlify/GitHub Pages in their DNS settings.
6
Troubleshooting
Model loading (503): HF server busy. Wait 30s and retry.

CORS errors: Some models have CORS restrictions. Try a different model.

Slow generation: Free tier is shared. Expect 30s–5min for video. Images: 5–30s.

No output: Check browser console (F12) for API errors.
7
Limits & Tips
• HF free tier: ~30 requests/hour without token
• With token: higher limits, priority queue
• Video models are slowest (GPU-heavy)
• FLUX Schnell is fastest for images
• MMS-TTS works instantly (CPU model)
• Results are not stored by HF — download to save
